# Product I18n Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add offline-filled multilingual product data and keep every customer-facing product response in the latest turn language without increasing live reply latency.

**Architecture:** Store product translations in a separate table keyed by product and normalized language. Load product translations together with products, pick localized fields at render time, and keep product matching aware of all translations. Fill translations through an offline script that uses the configured LLM provider and never runs inside the customer conversation path.

**Tech Stack:** FastAPI, SQLAlchemy async models, Python standard unittest, React TypeScript, existing LLM chat completion wrapper.
